# v0.10.0 — 무엇을 만들었나: 내용 기반 (요청·파일 → LLM)  (v0.10.0 · 2026-07-05)

## 한 줄 요약
git 커밋만이 아니라 **실제 프롬프트·다룬 파일을 읽고** "무엇을 만들었나"를 LLM이 서술한다.

## 왜

v0.9.0의 "무엇을 만들었나"는 **네가 쓴 git 커밋 제목**이었다. 사용자 질문: "실제 프롬프트·답변·만든
파일 내용을 보고 뭘 만들었는지 알려주는 거 아니냐 — 그렇게 돼 있나?" → **아니었다.** 기존 파이프라인은
`message.content`를 카운트만 하고 원시 텍스트를 버렸다. 이 버전이 그 방향을 연다.

## 무엇이 바뀌었나

- **`src/core/intent.ts`(신규):** 세션 JSONL에서 **user 요청문 + tool_use 파일**을 추출(`extractIntent`),
  KST 창별로 모으고(`collectIntents`), 마스킹 전 블록으로 조립(`buildIntentContext`),
  gitleaks 마스킹(`prepareIntentSend`, fail-closed).
- **`analyze --llm --send`:** `## 무엇을 만들었나 — 내용 기반 요약` 프로즈(마스킹된 요청·파일 → LLM).
  `--send` 없으면 dry-run이 "보낼 내용"을 그대로 출력(전송 0).
- **`createAnthropicBuiltSummarizer`:** "요청·파일에서 무엇을 만들었나 서술, 없는 성과 지어내지 말 것"
  시스템 프롬프트. env `AIMM_BUILT_MODEL`.

## 프라이버시 경계 (첫 원시-텍스트 외부 전송)

이건 프로젝트에서 **처음으로 원시 대화 텍스트가 (마스킹 후) 외부 LLM으로 나가는** 기능이다. 그래서 경계를 코드로 엄격히:

1. **격리:** 원시 텍스트는 `intent.ts` 안에서만 산다. `NormalizedSession`·digest·거울(systemMessage)·초상은
   절대 원시 텍스트를 보지 않는다(추출→마스킹→전송 후 폐기).
2. **파일 = basename만:** `C:\Users\jeonj\.claude\...` 같은 절대경로 대신 `render.ts`. 머신 구조·홈·타
   프로젝트 경로 노출 제거.
3. **노이즈 필터:** 시스템·스킬 주입 "user" 메시지(`<command-*>`·`<task-notification>`·`<system-reminder>`·
   `Base directory for this skill`·SKILL 덤프·`Unified diff`/`=== DIFF:`·리뷰 보일러플레이트)는 걸러 진짜
   요청만 남긴다.
4. **fail-closed dry-run:** `--send` 아니면 마스킹된 "보낼 내용"을 화면에 보여주고 전송 0. 보고 나서 결정.
5. **상한:** 프롬프트 60개·파일 80개·프롬프트 300자 — 노출면·토큰 비용 억제.
