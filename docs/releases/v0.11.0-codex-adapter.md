# v0.11.0 — Codex CLI 어댑터: 진짜 멀티-LLM 비용 비교 (Phase 2 · 2026-07-09)

## 한 줄 요약
Claude Code에 이어 **Codex CLI**를 소스로 추가 — 두 도구의 실제 토큰·비용을 나란히 비교한다.

## 왜

ROADMAP Phase 2("멀티 LLM, 로컬 로그가 있는 도구부터")의 첫 실현. Cursor(v0.5.0)는 토큰이
~4%만 채워지고 모델이 항상 "default"라 **비용 미상**(시간·빈도만)이었다. 이번엔 스파이크로
확인: Codex는 `~/.codex/sessions/**/rollout-*.jsonl`에 **모델·토큰이 실재**한다
(`event_msg/token_count.info.last_token_usage` + `turn_context.model`) → **providesCost=true**.
그래서 이 버전에서 처음으로 "Claude vs Codex" 도구 간 비용 비교가 성립한다.

## 무엇이 바뀌었나

- **`src/parse/codex.ts`(신규):** rollout 이벤트 스트림 파서(순수). `last_token_usage`를 TokenTotals로
  매핑(캐시 입력→cacheRead, reasoning→output), `turn_context.model`·`session_meta.cwd` 추적,
  도구를 공유 어휘로 정규화(`shell_command→Bash`, `apply_patch→Edit`, `update_plan→TodoWrite`) +
  명령 동사·파일 확장자 다이제스트. 라인 손상은 skip+warning(계약).
- **`src/adapters/codex.ts`(신규):** `~/.codex/sessions` 재귀 발견 + 파싱 → `SourceAdapter`.
  `providesCost=true`. Codex 미설치는 조용한 no-op.
- **`src/pricing.ts`:** GPT-5 계열(`gpt`) 단가 추가(추정). 캐시 배수는 공통.
- **`src/core/standup.ts`:** `ANALYSIS_ADAPTERS`에 `codexAdapter` 등록(claude + cursor + codex).
- **`src/core/render.ts`:** `shortModelName`에 Codex(gpt/codex)·Fable 추가 → 버전 여러 개를 한 줄로 병합.

## 경계·원칙 유지

- **비용 정직성:** 토큰을 신뢰 가능하게 주는 소스(Claude, Codex)만 비용 집계. Cursor는 미상 유지.
- **닫힌 어휘 다이제스트:** Codex도 원시 명령·경로를 저장하지 않고 도구명·동사 카테고리·확장자만.
- **어댑터 격리:** Codex 포맷 변동은 `parse/codex.ts` 한 곳만 고치면 됨.
