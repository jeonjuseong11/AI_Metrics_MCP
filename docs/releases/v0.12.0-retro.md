# v0.12.0 — 회고록(retro) + 주간 자동 생성  (2026-07-09)

## 한 줄 요약
사용 패턴과 "무엇을 만들었나"를 **한 회고 문서**로 묶고, **주간 자동 저장**까지 붙였다.

## 왜

두 요구: (1) 내 AI 사용 패턴 분석, (2) 뭘 했고 뭘 만들었는지 회고록. 둘 다 이미 코어에
있었다(`analyze`=패턴+상황, `intent`+`built`=무엇을 만들었나). 이 버전은 **새 엔진 없이**
그 둘을 회고 프레이밍으로 묶고, 주기 자동화를 얹는다.

## 무엇이 바뀌었나

- **`aimm retro`(신규 명령):** 사용 패턴 + 무엇을 만들었나를 한 문서로. 기간 기본 **최근 1주**
  (`--period month`도). `analyze`와 동일 엔진(멀티소스 Claude+Cursor+Codex, 전송 경계) +
  회고 헤더. `--repo`면 작업 성격·만든 것까지, `--llm --send`면 산문 회고.
- **`retro --write`(주간 자동):** 회고를 `~/aimm/retro-<end>.md`로 저장. **결정적**(LLM·키·전송 0),
  **주간 멱등**(같은 창 파일 있으면 skip, `--force`로 덮어씀), 실패 시 에러 노트(§4.4).
  OS 스케줄러(Windows schtasks / cron) 한 줄로 주 1회 호출 → 회고가 쌓임(README 문서화).
- **`src/core/retro.ts`(신규):** `runRetroWrite` — buildAnalysis(결정적) → renderAnalysis("AI 회고") → 파일.
- **`renderAnalysis` heading 파라미터:** 기본 "AI 사용 분석", retro는 "AI 회고"(하위호환).
- **`emitAnalysis` 공통화:** `analyze`/`retro`가 같은 몸통(build+무엇을만들었나+preview) 공유 — dup 아님.

## 재사용 (ponytail)

새 파서·분석·마스킹 0. 기존 `buildAnalysis`/`renderAnalysis`/`intent`/`situation` 조립 +
얇은 회고 프레이밍·파일 영속. `retro`는 `analyze`에 (a)기간 기본 (b)회고 헤더 (c)파일 저장만 더한 것.
