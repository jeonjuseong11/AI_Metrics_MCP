# v0.13.0 — 회고 강화: memoir 내레이터 + retro MCP 도구  (2026-07-09)

## 한 줄 요약
`retro --send`가 조각 섹션이 아니라 **한 편의 회고 글**로, `retro`가 **MCP 도구**로 Claude Code 안에서.

## 무엇이 바뀌었나

- **memoir 내레이터:** `retro --send`가 이제 회고 톤 산문(기간 도입 → 무엇을 만들었나 → 사용 리듬·모델·시간대 → 마무리)을 낸다. `createAnthropicMemoirNarrator`(env `AIMM_MEMOIR_MODEL`). `buildNarrativeContext`가 이미 모든 재료(모델·시간대·프로젝트·만든것·작업성격·작업내용)를 한 사실 블록으로 모으고 있어 — **새 데이터 파이프 0, 프롬프트 1 + 배선 1.** `analyze --send`는 그대로 주간 요약 톤.
- **retro MCP 도구:** `standup`·`analyze`에 이어 `retro` 노출. Claude Code가 `retro(period|start/end, repo, author)`로 회고 문서를 결정적으로 받는다(전송 없음). `tools/list` 확인.

## 재사용 (ponytail)
둘 다 기존 조립 위 얇은 층. memoir=프롬프트 스왑(내레이터 DI), MCP retro=`buildAnalysis` + 회고 헤더 재사용.

## 한계
memoir 프로즈는 `ANTHROPIC_API_KEY` 필요 — 이 릴리스는 배선·타입만 검증(프로즈 실행검증은 키 있을 때). 결정적 회고(`retro` LLM 없이)·MCP 도구는 실측 확인.
