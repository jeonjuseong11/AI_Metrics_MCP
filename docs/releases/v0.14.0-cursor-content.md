# v0.14.0 — Cursor 내용 다이제스트 + 내용 격리 해제  (2026-07-09)

## 한 줄 요약
Cursor의 **요청(user 프롬프트)**이 이제 "무엇을 했나" 롤업에 포함된다 — 비용은 계속 격리.

## 왜

멀티소스가 됐지만(v0.5.0 Cursor, v0.11.0 Codex) Cursor는 **비용 미상**이라 "무엇을 했나"
내용 롤업에서도 통째로 제외돼 있었다(v0.6.0 격리). 하지만 **"무엇을 했나"는 활동 서술이지
비용이 아니다** — Cursor로 뭘 했는지도 정직하게 보여야 한다. 이 버전이 그 격리를 **내용에
한해** 푼다(비용·모델·시간 롤업은 여전히 cost-known만).

## 무엇이 바뀌었나

- **`adapters/cursor.ts`:** Cursor 버블 `type===1`(user) 텍스트를 세면 `SessionContentDigest.userPrompts`.
  (버블 구조 실측: type1=user·type2=AI. 도구·파일 신호는 AI 버블에 있으나 노이즈라 후속 — 현재 **요청 건수만**.)
- **`core/analysis.ts`:** `contentSummary`를 cost-known 세션만이 아니라 **전 소스** digest로 집계
  (내용 격리 해제). 비용·모델·시간·프로젝트 롤업은 그대로 cost-known만.
- **caveat 정정:** "요청 건수는 전 소스(Cursor 포함), 도구·영역·명령은 Claude Code만."

## 정직성

- Cursor는 **요청 건수만** 기여(도구·영역·명령 카테고리 아님) — 부분 신호임을 caveat에 명시.
- 비용·모델은 여전히 Cursor 제외(providesCost=false) — cost 오염 없음. `hasUnknownModel` 불변.

## 테스트

기존 "cost-unknown 내용 제외" 회귀 테스트를 **"포함"으로 반전**(격리 해제 의도 명시 + 비용 격리 유지 검증).
257 그린. 실측: `analyze`가 Cursor 요청 포함(userPrompts↑), 비용줄 무변.
