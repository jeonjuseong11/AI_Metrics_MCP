# v0.15.0 — 커밋 × AI 세션 시간 상관 (비용 귀속 아님)  (2026-07-09)

## 한 줄 요약
"이 커밋 즈음 AI 세션이 있었나"를 **시간 겹침**으로 보여준다 — 커밋별 비용 귀속은 하지 않는다.

## 왜 (그리고 왜 비용이 아닌가)

세션↔커밋을 잇자는 요청이 있었지만, **커밋별 비용 귀속은 이 프로젝트가 v0.9.0에서
의도적으로 안 한** 것이다: 한 세션이 여러 커밋·몇 시간에 걸쳐 있어 특정 커밋에 $X를
붙이는 건 **거짓 정밀도**이고, 프로젝트 척추("서술이지 평가·수치 지어내기 아님")를 어긴다.

그래서 *서술로 안전한 것만* 한다 — 커밋 시각이 어떤 AI 세션 시간창 안에 들었는지(**시간
상관**). "겹쳤다"이지 "이 세션이 이 커밋을 만들었다"(인과)가 아니다.

## 무엇이 바뀌었나

- **`src/core/correlate.ts`(신규):** `correlateCommits(commits, sessions, ±30분)` — 커밋 시각이
  세션 `[start,end]±pad` 안에 드는 비율. 순수·결정적.
- **`buildAnalysis`:** `--repo` 주어지면 `correlation` 계산(커밋·세션 둘 다 이미 수집됨).
- **`renderAnalysis`:** `## 커밋 × AI 세션 (시간 상관)` 섹션 — "커밋 M건 중 N건이 세션
  시간대와 겹침 (X%)" + **비용 귀속·인과 아님** 라벨. MCP `retro`·CLI `analyze --repo`에 노출.

## 정직성

- 비용 귀속 0. 인과 단정 0. 시간 겹침(±30분)이라는 상관만.
- 겹침 판정에 세션 소스 무관(Claude·Codex·Cursor 전부 = AI 활동 존재).
