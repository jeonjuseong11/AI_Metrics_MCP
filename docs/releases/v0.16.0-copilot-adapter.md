## 한 줄 요약
GitHub Copilot(VS Code Chat)이 소스로 추가 — 시간·요청·모델명(비용 미상, Cursor식).

## 왜 / 데이터 실태

멀티-LLM 확장 요청. Copilot 로그 실측(2026-07-09): VS Code가 대화를
`<Code>/User/workspaceStorage/<hash>/chatSessions/*.json`에 둔다. 세션에 `requests[]` =
{`message.text`(프롬프트), `timestamp`, `modelId`(예 `copilot/gpt-5-mini`)}. **토큰은 없음**
→ Cursor처럼 **providesCost=false**(시간·요청·모델명만, 비용·모델 롤업엔 안 낌).

## 무엇이 바뀌었나

- **`adapters/copilot.ts`(신규):** `workspaceStorage/*/chatSessions/*.json` 발견 + 파싱.
  `parseCopilotSession`(순수): requests → 메시지(model=modelId, 토큰 0) + `userPrompts`(요청 수).
  세션 시각 = 요청 타임스탬프(없으면 creation/lastMessage). 파일 부재/손상 fail-soft.
- **`ANALYSIS_ADAPTERS` 등록:** claude · cursor · codex · **copilot**.

## 정직성

- **비용·모델 롤업 제외**(providesCost=false) — Copilot의 gpt 모델이 비용 믹스를 오염시키지 않음.
- "도구별 사용"(시간·세션 수) + "무엇을 했나"(요청 건수)에만 등장. 도구·파일 신호는 후속.
