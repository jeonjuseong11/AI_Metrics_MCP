# v0.18.0 — `--via-claude`: 구독(claude 로그인)으로 산문, API 키 불요  (2026-07-09)

## 한 줄 요약
`--send`의 LLM 산문을 API 키(종량제) 대신 **구독**(`claude --print`)으로 낼 수 있다.

## 왜

산문(회고·주간 서술·무엇을 만들었나)은 `ANTHROPIC_API_KEY`(Anthropic SDK, 종량제)가
필요했다. 하지만 이미 Claude Code **구독**을 쓰는데 또 API 키를 넣는 건 낭비다. `--via-claude`는
AIMM이 SDK 대신 `claude --print`(Claude Code CLI = 구독 인증)를 자식 프로세스로 호출한다.
키 0, 구독 그대로.

## 무엇이 바뀌었나

- **`src/llm/claudeCli.ts`(신규):** `claude --print`에 프롬프트를 stdin으로 넘겨 산문을 받는
  Summarizer들(`createClaudeCliSummarizer`/`Narrator`/`MemoirNarrator`/`BuiltSummarizer`).
  **같은 시스템 프롬프트**(anthropic.ts에서 export)를 재사용 → SDK 경로와 톤 일치.
- **`--via-claude` 플래그:** `standup`·`analyze`·`retro`의 `--send`에 붙이면 SDK 대신 구독 경로.
- **폴백 유지(§4.4):** 미로그인·실패는 `SummarizerError` → 결정적 문서로 자동 폴백(크래시 없음).

## 사용

```bash
# API 키 없이, 구독으로 회고 산문
node dist/cli.js retro --period month --repo . --llm --send --via-claude
node dist/cli.js analyze --repo . --llm --send --via-claude
```

`claude` CLI가 로그인돼 있어야 함(안 돼 있으면 `claude` 한 번 실행해 로그인). 미로그인 시
"claude 미로그인" 경고 + 결정적 문서로 폴백.

## 한계

- 산문은 여전히 *서술이지 평가 아님*(척추 유지) — 모델만 SDK→CLI로 바뀜.
- 검증: 스모크(미로그인 → 우아한 폴백)까지 확인. 실제 산문은 로그인된 `claude`가 있는 터미널에서.
