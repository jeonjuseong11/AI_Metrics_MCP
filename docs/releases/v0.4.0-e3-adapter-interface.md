# v0.4.0 — E3 멀티소스 어댑터 인터페이스

릴리스: 2026-06-17 (KST)

## 한 줄 요약
AI 사용 소스를 `SourceAdapter` 계약 뒤로 격리 — 오케스트레이터가 Claude Code 파일 구조를 직접 모르게 하고, 새 소스(E5 Cursor)를 "인터페이스 구현 + 주입"만으로 드롭인할 토대를 만든다.

## 무엇이 바뀌었나 (구조 작업 — 사용자 체감 기능 변화 없음)
- 신규 `src/adapters/types.ts`: `SourceAdapter`(`id`·`displayName`·`collect()`) + `CollectOptions`(`paths`·`rootDir`) 계약.
- 신규 `src/adapters/claudeCode.ts`: 기존 발견(`fs/discover`)·읽기(`fs/sessions`)·파싱(`parse/claudeCode`)을 합성한 `claudeCodeAdapter`.
- `src/core/standup.ts`: `buildStandup`/`buildAnalysis`가 주입 가능한 `adapter`(기본 `claudeCodeAdapter`)에 의존.

## 코드 before/after
**before** — 오케스트레이터가 소스를 직접 알았다:
```ts
const files = opts.sessionFiles ?? (await discoverSessionFiles(opts.projectsDir));
const parsed = await readSessionFiles(files);
```
**after** — 계약 뒤로 격리:
```ts
const adapter = opts.adapter ?? claudeCodeAdapter;
const collectOpts: CollectOptions = {};
if (opts.sessionFiles) collectOpts.paths = opts.sessionFiles;
if (opts.projectsDir) collectOpts.rootDir = opts.projectsDir;
const parsed = await adapter.collect(collectOpts);
```
- **CLI·hook·MCP 무변경** — 전부 `adapter` 미지정 → 기본 어댑터가 흡수. 출력·동작 동일(스모크: `analyze`가 기존과 같은 문서 산출).
- 빈 배열 불변식 보존: `sessionFiles: []`는 자동 발견을 건너뛰고 세션 0(`[] ?? discover()`가 `[]` 유지). 다수 기존 테스트의 전제.
- `portrait.ts`: "도구별 사용" 표의 stale 렌더 출력 문구 "(E3/E5)"→"(E4)" 1줄 정정(테스트로 고정; 라벨 일반화는 source 태깅 필요 → E4 일임).

## 범위 경계 (CEO 플랜 "제네릭 IF만")
- 계약은 **순수 구조만**. 능력 모델(`providesCost`·토큰 부재 처리)·source 태깅·레지스트리·다중 어댑터 집계는 **범위 밖**(E5/E4).
- 어댑터는 **Claude Code 하나만**. Cursor 어댑터는 E5(스파이크 먼저).
