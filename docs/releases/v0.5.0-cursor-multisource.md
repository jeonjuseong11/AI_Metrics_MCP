# v0.5.0 — Cursor 어댑터 + 멀티소스 통합 (E5 + E4-lite)

릴리스: 2026-06-17 (KST)

## 한 줄 요약
Cursor를 두 번째 AI 사용 소스로 추가 — analyze/portrait "도구별 사용" 표에 Cursor 행(비용 미상)이 뜬다.

## 무엇이 바뀌었나
- **스파이크 게이트(2026-06-17)**: Cursor 로컬 SQLite(`state.vscdb`)는 메시지별 `createdAt`은 100% 있으나 토큰은 ~4%만·모델은 전부 `"default"` → **시간·빈도만, 비용 미상** 확정.
- 신규 `src/adapters/cursor.ts`: `state.vscdb`의 `cursorDiskKV` 버블을 composer(대화)별 세션으로(토큰 0). `node:sqlite` 내장 사용(새 npm 의존성 0).
- `SourceAdapter.providesCost`(E3에서 미룬 능력 모델, 이제 실소비자 등장) + `NormalizedSession.source` 태깅.
- **cost-unknown 소스 격리**: `providesCost:false` 소스(Cursor)는 모델/비용/시간 롤업·`hasUnknownModel`에 안 들어가고 **byTool에만** 기여 → Claude Code 출력 무오염.
- analyze/portrait는 멀티소스(cli/mcp가 `ANALYSIS_ADAPTERS` 주입), **standup·hook은 Claude Code 단독 유지**(스크럼 초안 토큰중심 보존).

## 출력 before/after
**before** — portrait "도구별 사용"이 `| Claude Code | N |` 한 행 하드코딩 + "후속 단계(E4)" 안내.

**after** — byTool로 실제 도구별 행(이 레포 실데이터, 2026-01~06):
```
## 도구별 사용
| 도구 | 세션 | 추정 비용 |
|------|------|-----------|
| Claude Code | 26 | 약 $3,981.92 |
| Cursor | 3 | 미상 |
_세션 정의는 도구마다 다릅니다: Claude Code=세션 로그, Cursor=대화(composer)._
```
- Cursor 비용은 **"미상"**(서버측 회계라 로컬 토큰 불완전·모델 미상). "세션 정의 다름" 캐비엇으로 단순 비교 오인 방지.

## 검증
- 테스트: **133 → 146 그린**(신규 `cursor`(7)·`multisource`(5) + portrait byTool(1); `adapters` E3 DI-seam은 `adapters:[]`로 마이그레이션). 오염 6차원(byModel·byDay·byHourKst·byProject·totals·hasUnknownModel) 전부 격리 단언.
- **오염 차단 회귀 테스트**: Cursor의 unknown 모델·0토큰이 `hasUnknownModel`을 켜지 않음 / 모델믹스에 unknown 행 없음 / 출력에 "단가 미상" 없음.
- **결정성**: 코어 기본 claude-only + spy로 `cursorAdapter.collect` 미호출 증명(기존 테스트 실 Cursor DB 미접촉). 멀티소스는 cli/mcp가 명시 주입.
- 설계는 **적대적 spec 리뷰(blocker 5건 반영) + 구현 코드 리뷰**로 검증. `node:sqlite`는 `createRequire`+`import type`로 번들러 우회, 부재 DB는 조용히 skip(경고 0). `tsc --noEmit`·`npm run build` 클린(@types/node ^22, engines>=22).
