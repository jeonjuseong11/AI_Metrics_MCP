# v0.7.0 — SessionStart 거울  (v0.7.0 · 2026-06-28)

## 한 줄 요약
Claude Code를 열 때마다 어제·이번주 AI 사용 현황이 한 줄로 자동 표시된다 — 트리거 없이도 매일 확인.

## 무엇이 바뀌었나

**트리거 부재 → 매일 거울:** AIMM은 SessionEnd hook으로 초안을 만들지만, 그 초안을 보려면 사용자가 직접 파일을 열어야 했다. 하루도 열지 않으면 AIMM은 조용히 죽는다. v0.7.0은 "Claude Code를 여는 순간"이라는 트리거를 붙인다 — 아무것도 안 해도 그날의 현황을 하루에 한 번 마주하게 된다.

- **`aimm session-start` CLI(신규):** SessionStart hook 진입점. stdin JSON `{"source":"startup"|"resume"}` 를 읽고, `startup`·`resume` 시에만 어제+최근7일 비용-확인 현황을 `top-level systemMessage`로 출력한다. `compact`·`clear`는 스킵. 항상 exit 0(실패 안전).
- **`renderGlance` 포매터(신규):** 세 변형을 결정적으로 포매팅한다.
  - **정상:** `🪞 어제 N세션 · $X.XX · 이번주(최근7일) N세션 · $X.XX`
  - **어제-없음:** `🪞 어제 기록 없음 · 이번주(최근7일) N세션 · $X.XX`
  - **cold-start:** `⚠️ aimm: 세션 데이터 없음(cold-start)`
- **`aimm init` SessionStart 멱등 등록(확장):** 기존 SessionEnd 등록에 더해 SessionStart hook도 병합. 별개 sentinel 마커를 써서 SessionEnd와 교차되지 않는다.
- **헬퍼 hoist:** `weekdayOf` / `WEEKDAY` / `isoDatePlusDays`를 `patterns.ts`에서 `core/day.ts`로 이동 — 여러 모듈이 재사용.

## 핵심 정정 사항 — top-level systemMessage

가시성 스파이크에서 발견: Claude Code SessionStart hook 출력은 **top-level `systemMessage`** 필드에 있어야 사용자에게 보인다. `hookSpecificOutput.systemMessage`에 넣으면 Claude Code가 표시하지 않는다(조용히 무시). `toHookOutput()` 유틸리 함수는 항상 `{ systemMessage: "..." }` 형태로 내보낸다.

## 산출물 before/after

**before** — session-start CLI 없음. Claude Code 시작 시 AIMM 정보 전무. 사용자가 직접 `aimm analyze`·`aimm today`를 실행해야만 현황 확인 가능.

**after** — Claude Code 시작(startup/resume) 시 자동 표시(2026-06-28 실제 실행):

```bash
$ echo '{"source":"startup"}' | node dist/cli.js session-start
{"systemMessage":"🪞 어제 기록 없음 · 이번주(최근7일) 15세션 · $378.25"}
```

> 위 출력은 어제-없음 변형(주말·휴일 다음 날 등 어제 세션 0인 경우). 닫힌 어휘만 포함 — 경로 구분자(`/`·`\`)·원시 명령·프롬프트 텍스트 없음.

## 검증

- 테스트: **178 → 204 그린** (신규 26개 — `render-glance` 4 · `session-start` 4 · `session-start-run` 3 · `cli-session-start` 4 · `day` +9 · `init` +2)
- `tsc --noEmit` 클린(strict · exactOptionalPropertyTypes).
- privacy 단언: `systemMessage`에 `/`·`\` 부재 + raw 토큰 0.
- source 필터 단언: `compact`·`clear` 전달 시 exit 0 + 빈 출력.
- 가시성 스파이크 확인: top-level `systemMessage` 필드가 Claude Code에 표시됨을 직접 확인. `hookSpecificOutput.systemMessage` 중첩 구조는 미노출 → 코드에 반영.
- `aimm init --dry-run`: SessionStart 항목이 별개 마커로 추가되고 SessionEnd 항목과 충돌 없음.

## 알려진 한계

- **resume 포함**: 현재 `startup`·`resume` 둘 다 거울을 표시한다. `resume`이 빈번하면 장황할 수 있음 — 후속에서 `startup`만으로 좁힐 수 있다.
- **cold-start 표시**: 세션 데이터가 전혀 없는 머신(신규 설치 직후)에서는 `⚠️ aimm: 세션 데이터 없음(cold-start)` 를 낸다.
