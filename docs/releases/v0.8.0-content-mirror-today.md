# v0.8.0 — 내용 있는 거울 + `aimm today`  (v0.8.0 · 2026-07-05)

## 한 줄 요약
거울 한 줄이 "얼마나"(비용)에서 "무엇을"(다룬 영역)로 바뀌고, 세션 밖에서도 `aimm today`로 현황을 조회한다.

## 무엇이 바뀌었나

**활동축 → 영역축(거울이 날짜를 구별):** v0.7.0 거울은 활동축(`구현·탐색`)을 보여줬는데,
거의 모든 코딩일이 똑같이 나와 "오늘 내가 이걸 썼구나" 순간이 안 생겼다. v0.8.0은 날짜를
구별하는 **영역축 share%**(다룬 파일 종류)로 교체한다.

- **거울(신규 축):** `🪞 어제: 3세션 · $12.40 · TypeScript 60%·문서 25% | 이번주(최근7일): 15세션 · $378.25`
  상위 2개 영역 share%(count/영역합). 영역 없으면 접미 생략(NaN 가드).
- **`aimm today`(신규 CLI):** 세션 밖에서 아무 때나. 오늘(지금까지)은 3축(활동·영역·명령) 풀뷰,
  어제는 한 줄, 이번주(최근7일, **오늘 포함**)는 요약. claude-only(cost-known).
- **`renderContentBlock` 추출(DRY):** analyze의 "무엇을 했나" 3축 블록을 공유 함수로 뽑아
  analyze 렌더와 `aimm today`가 재사용. 헤딩은 호출부가, 정직성 캐비엇은 블록 안(단일 소스).

## parse-once + 수집창 캐핑

**`collectSessions` seam:** `buildAnalysis`에서 세션 수집을 `collectSessions(common, adapters)`로
분리. `analyze()`가 이미 인메모리로 range 필터 + cost-unknown 격리를 하므로, 거울/today는
세션을 **1회 수집**하고 `analyze()`를 범위별로 여러 번 부른다(디스크 파싱 1회).

- `buildAnalysis` 시그니처 불변 → standup/analyze/portrait 4진입점 무영향.
- **`CollectOptions.sinceMtimeMs`** 성근 프리필터: 자동 발견 파일을 최근-창 mtime으로 좁혀
  SessionStart startup 비용을 O(전체 히스토리)에서 상수(최근 ~10일)로. 명시 paths는 존중(무필터).
  정확한 KST 날짜 필터는 여전히 `analyze()`가 인메모리로 — mtime창(10일) ⊇ 분석창(≤8일)이라 안전.
- 거울 startup 파싱 2 → 1회.

## 알려진 한계

- **거울 vs today 이번주 차이**: 거울 이번주는 ending-어제(startup=아침 기준), today 이번주는
  ending-오늘(오늘 포함). 같은 날 두 수가 다를 수 있음 — 각자 내부적으로 정확(설계 의도).
- **`aimm today`도 claude-only**: cost-unknown 소스(Cursor)의 내용은 미파악이라 제외. 멀티소스
  today는 후속(renderContentBlock 캐비엇 파라미터화 필요).
