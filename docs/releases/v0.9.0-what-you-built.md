# v0.9.0 — 무엇을 만들었나 (비용 ↔ 성과)  (v0.9.0 · 2026-07-05)

## 한 줄 요약
"어떻게 썼나"(도구 빈도)를 넘어 **"이 비용으로 무엇을 만들었나"**(git 커밋 성과)를 보여준다.

## 무엇이 바뀌었나

**행위 → 성과:** 기존 "무엇을 했나"는 tool_use 빈도(탐색42%·TypeScript73)라 *행위*였다.
사용자는 *성과* — "$X로 이런 걸 만들었다" — 를 원했다. git 커밋 제목이 그 신호다.

- **`summarizeSituation`이 `built` 추출:** feat/fix/refactor/perf 커밋 **제목**(제품을 빚는 타입)을
  입력 순서(최신 우선) 유지해 뽑는다. docs/chore/style/ci/build/test/revert/기타는 개수로만.
  상한 `MAX_BUILT`(8), 초과분은 "외 N건".
- **`analyze --repo` 신규 섹션:** `## 무엇을 만들었나 (이 기간 추정 $X)` — 만든 것(커밋 제목) 나열 +
  같은 기간 세션 총액. "작업 성격(타입 분포)" 섹션 앞에 온다.
- **`aimm today --repo`:** 오늘(KST) 커밋을 🔨로 오늘 섹션에 표시. git 실패는 조용히 무시(현황은 계속).
- **LLM 층(`--llm`):** narrative에 `[만든것]` 사실줄 추가 → LLM이 성과를 산문화("이번 주엔 거울을
  내용화하고 today를 만들었다"). **마스킹 경계 통과 후에만** 전송(gitleaks fail-closed).

## 프라이버시 스코프 (핵심)

커밋 제목은 **원시 텍스트**다. 그래서 노출 위치를 엄격히 나눈다:

| 위치 | 원시 커밋 제목 | 이유 |
|------|:---:|------|
| `analyze --repo` (로컬 문서) | ✅ | 본인이 보는 로컬 문서 · 자기 git 로그 |
| `aimm today --repo` (로컬 CLI stdout) | ✅ | 로컬 터미널 출력 |
| `--send`(LLM 전송) | 마스킹 후 | gitleaks fail-closed 경계 통과 |
| **거울(systemMessage)** | ❌ | Claude Code에 표출 — 닫힌 어휘만 |
| **초상(공유 산출물)** | ❌ | 공유용 — 프로젝트명도 숨기는 추상 뷰 |

## 알려진 한계

- **비용 귀속 불가:** "$X로 커밋 Y" 가 아니라 "이 기간 총 $X · 이 기간 만든 것들". 어느 세션이
  어느 커밋을 만들었는지는 매칭하지 않음(신뢰 불가라 미제공).
- **repo 필요:** "무엇을 만들었나"는 git 커밋 기반. repo 없으면 안 나옴(today는 --repo 명시).
