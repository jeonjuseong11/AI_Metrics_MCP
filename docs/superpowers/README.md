# 설계·계획 기록 — AI 페어링 과정

이 폴더는 AIMM을 **Claude Code와 페어로** 설계·구현한 과정의 1차 자료다.
구현 코드가 "무엇을"이라면, 여기는 **"왜 그렇게 결정했나"의 근거**다.

- [`aimm-ceo-plan.md`](./aimm-ceo-plan.md) — 전략·스코프 확정.
  적대적 리뷰 1라운드, E1~E7 결정표, 영구 제외선(경영진 감시 모드).
- [`specs/`](./specs/) — 기능별 설계(목적·정체성·트레이드오프).
- [`plans/`](./plans/) — task 단위 구현 계획(brainstorm → spec → plan → 구현).

> 모든 설계·스코프 결정은 사용자가 내렸고, AI는 구현·초안 작성의 페어로 썼다.
