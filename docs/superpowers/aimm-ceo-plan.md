## Vision

### 10x check — what the user feels
지금 AIMM은 "Claude Code 사용을 정직하게 서술하는 CLI"다. 10x 버전은 도구가 아니라
거울이다: 당신이 AI와 어떻게 일하고 성장하는지를 보여주는 살아있는 초상. 숫자
대시보드가 아니라 craft의 내러티브. "이번 달 너는 막힌 디버깅엔 밤에 Opus를, 빠른
UI엔 오후에 Cursor를 잡았다. fix 커밋은 AI 디버깅이 몰린 다음 날 몰린다." 셀프
리뷰·승진 패킷에 그대로 붙이는, 네가 100% 통제하는(로컬·마스킹·네 것) 증거.

### Platonic ideal
PC에서 쓰는 모든 코딩 LLM을 자동 발견해 "무엇을 어떤 상황에 쓰는지"를 한 장의
정직한 초상으로. 단 한 줄도 감시가 아니다. 전부 본인만, 본인을 위해.

### 변하지 않는 척추
개인 상향식 · 로컬·마스킹 우선 · 서술이지 평가 아님 · 결정적 메트릭은 LLM 우회 ·
경영진 감시 모드는 끝까지 제외.

## 어댑터 경계 (E3 / E4 / E5 — 과빌드 방지)

- **E3** = 어댑터 계약(추상 베이스) + **Claude Code 어댑터 하나만**. 제네릭 인터페이스만 세움.
- **E5** = 그 계약을 구현하는 **Cursor 어댑터**(별개 작업).
- **E4** = E3+E5 둘을 **소비**하는 통합 뷰. E3을 "모든 어댑터까지" 과빌드하지 말 것.

## E1 완료 기준 (trivial-done 방지)

1.0 = 마크다운, **텍스트+표만(차트/그래프 없음)**. 필수 필드: ① 도구별 세션 수
② 비용 요약 ③ E2 인사이트 1개 이상 ④ 시간대 패턴 한 줄 ⑤ 본인 메모 자리. PDF는
E2 이후 옵션 플래그.

## E2 시작 상관 가설 (수락 = 빈 약속 방지)

1차 2쌍 고정: **(a) AI 디버깅 몰린 세션 ↔ 다음날 fix 커밋 증가**, **(b) 평소 대비
비용 급증일**. 전부 "상관이며 인과·증명 아님" 라벨 필수. 추가 가설은 별도 brainstorm.

## E6 가드레일 (코히어런스 — 단언이 아니라 구조)

E6은 메모리의 제외 항목 "경영진 감시(2b)"와 **구조적으로 다르다**: 공유 가능한
식별자 없이 *본인의 시간 정황만* 캡처하고, 밖으로 나가는 산출물(E1 초상)은 raw
덤프가 아니라 **본인이 큐레이션한 것**이다. 2b는 위→아래 가시성이라 영구 제외, E6은
아래→본인. 조건:
- **default=OFF**, 명시적 config 플래그로만 활성(옵트인 메커닉).
- **시간·빈도만**, 내용/제목 캡처 금지. **최소 단위 = 시간(hour) 버킷, 초 단위 샘플링 금지.**
- 보존: **로컬 SQLite 90일 롤링 + `purge` 명령**, 업로드 영구 없음.
- "앱 켜놨다 ≠ 썼다" 항상 라벨.
- 플랫폼: **Windows 1.0**, macOS 2.0(접근성 권한 UX 포함), Linux/Wayland 미정(무한 공수라 보류).

## 권장 시퀀싱 (가치 우선 + 의존성)

```
가치 증명 ──────▶ 멀티소스 토대 ────▶ 통합 ──▶ 폼팩터/큰표면(독립)
 E1 → E2          E3 → E5(스파이크)    E4       E7(스파이크後) | E6(맨끝,가드레일)
```

1. **E1 초상** → 2. **E2 인사이트** → 3. **E3 어댑터 IF** → 4. **E5 Cursor(스파이크 먼저)**
→ 5. **E4 통합 뷰** → 6. **E7 대시보드** → 7. **E6 OS 캡처**.
- E7과 E6은 이 단계에서 **상호 독립**(E7은 E4 후 착수 가능, E6 완료를 안 기다림).
- 각 항목은 자체 brainstorm → spec → plan → 구현 사이클. 이 리뷰는 전략·스코프만 확정.

## 착수 게이트(스파이크) — ACCEPTED지만 먼저 검증

- **E5 Cursor 스파이크**: ≤1일 타임박스. 산출 = state.vscdb에 토큰/비용 유무 + **버전 간
  스키마 안정성 이력** 1단락. 분기: 토큰 有 → 풀 어댑터(비용 포함) / 無 → 시간·빈도만,
  E4 비용 축은 "미상". 스키마 불안정 → 버전-감지 레이어 추가.
- **E7 폼팩터 스파이크**: E4 완료 후 ≤1-2일. Tauri는 **Rust+WebView 새 툴체인**(크로스플랫폼
  패키징·자동업데이트) 점프임을 인지. 기존 엔진을 Tauri에서 소비 가능한지 확인 후
  착수. Electron/웹앱/강화 CLI도 후보로 비교.
