# Cursor 어댑터 + 멀티소스 통합 (E5 + E4-lite) — Design

Generated by /brainstorming on 2026-06-17 · **적대적 spec 리뷰 1라운드로 굳힘**(blocker 5건 + major 다수 반영)
Branch: main | Repo: AI_Metrics_MCP | 버전: v0.5.0

> Cursor를 두 번째 AI 사용 소스로 추가한다. 스파이크 결과 Cursor는 **토큰/비용은 못 주지만 시간·빈도는 완전**.
> 그래서 **analyze/portrait의 '도구별 사용' 표에 Cursor 행(비용 미상)**을 띄운다(MVP). 단 cost-unknown 소스는
> 기존 토큰/모델/비용 집계를 **오염시키지 않고 byTool에만** 기여한다. standup/hook은 Claude Code 단독 유지.

## 1. 스파이크 결과 (2026-06-17 실측 — 설계 근거)

`%APPDATA%/Cursor/User/globalStorage/state.vscdb`(SQLite; `ItemTable` + `cursorDiskKV`):
- **토큰 사용 불가**: 버블 `tokenCount`가 1423개 중 0 아닌 건 59개(4%), 5일 구간뿐. **모델 없음**(전부 `"default"`) → 비용 산출 불가. 진짜 회계는 서버측(`usageUuid` 참조).

- 게이트 분기 = **시간·빈도, 비용 미상.**

## 4. 계약 변경 — `src/adapters/types.ts`
```typescript
export interface SourceAdapter {
  readonly id: string;
  readonly displayName: string;
  /** 토큰/비용을 신뢰 가능하게 제공하는가. false면 분석이 비용을 "미상"으로 표기하고 모델/비용 집계에서 제외. */
  readonly providesCost: boolean;
  collect(opts?: CollectOptions): Promise<ParseResult>;
}
```
(`CollectOptions` 불변. E3 스펙 "능력 모델은 실소비자 생기면 추가" 예고대로 — cursor가 소비자.)

## 5. 모델 변경 — `src/types.ts`
`NormalizedSession`에 `source?: string` 추가. 오케스트레이터가 collect 후 `s.source = adapter.id`로 스탬프(어댑터 collect는 순수 유지). `metrics` 명령(직접 parse 경로)은 source 미지정 → 그 경로는 analyze를 안 타므로 `(unknown)` byTool 행은 현재 도달 불가(향후 주의).

## 6. Cursor 어댑터 — `src/adapters/cursor.ts` (신규)
```typescript
export const cursorAdapter: SourceAdapter = {
  id: "cursor", displayName: "Cursor", providesCost: false,
  async collect(opts = {}) {
    const warnings = []; const sessions = [];
    const dbPaths = opts.paths ?? [join(opts.rootDir ?? defaultCursorGlobalStorage(), "state.vscdb")];
    for (const dbPath of dbPaths) {
      if (!existsSync(dbPath)) continue;                 // ⚠️ 부재는 조용한 no-op(경고 없음) — Cursor 미설치 정상
      try {
        const { DatabaseSync } = await import("node:sqlite");   // lazy; ExperimentalWarning은 stderr-only
        const db = new DatabaseSync(dbPath, { readOnly: true });
        const rows = db.prepare("SELECT key, value FROM cursorDiskKV WHERE key LIKE 'bubbleId:%'").all();  // 읽기까지 try 안
        db.close();
        // ... 그룹핑(아래) ...
      } catch (e) { warnings.push({ line: 0, reason: `Cursor DB 읽기 실패(${dbPath}): ${e.message}` }); }  // 테이블 부재·락·손상 전부 여기
    }
    return { sessions, warnings };
  },
};
```
**그룹핑(throw 없음):**
- 각 row: `value`를 **BLOB 방어 디코드** — `const txt = typeof value === "string" ? value : Buffer.from(value).toString("utf-8");` → `JSON.parse(txt)`(실패 시 skip+warning).
- **키 가드**: `const parts = key.split(":"); if (parts.length < 3 || !parts[1]) { skip+warning; continue; }` → `composerId = parts[1]`.
- `createdAt`(ISO) 파싱(없거나 NaN → skip+warning). composerId별로 묶어 세션 1개: `sessionId=composerId`, `projectPath=undefined`, `messages`=버블당 `{model:"unknown", timestamp:createdAt, tokens:{0,0,0,0}}`, `startTime/endTime`=createdAt min/max.
- `defaultCursorGlobalStorage()`: win32=`%APPDATA%/Cursor/User/globalStorage`, darwin=`~/Library/Application Support/Cursor/User/globalStorage`, linux=`~/.config/Cursor/User/globalStorage`.

## 7. 멀티소스 수집 + 기본 어댑터 — `src/core/standup.ts`
`StandupOptions`·`AnalysisBuildOptions`: `adapter?: SourceAdapter`(E3) **제거 → `adapters?: SourceAdapter[]`**.
```typescript
export const ANALYSIS_ADAPTERS: SourceAdapter[] = [claudeCodeAdapter, cursorAdapter];

// buildStandup·buildAnalysis 공통 수집:
const adapters = opts.adapters ?? [claudeCodeAdapter];   // 코어 기본 = claude-only(테스트 결정적·footgun 없음)
const collectOpts: CollectOptions = {};
if (opts.sessionFiles) collectOpts.paths = opts.sessionFiles;     // 빈 배열 truthy 보존(E3 불변식)
if (opts.projectsDir) collectOpts.rootDir = opts.projectsDir;
const parsedList = await Promise.all(adapters.map(async (a) => {
  try {
    const r = await a.collect(a.id === "claude-code" ? collectOpts : {});  // 레거시 옵션은 CC에만; 타 소스는 자기 기본 발견
    return { sessions: r.sessions.map((s) => ({ ...s, source: a.id })), warnings: r.warnings };  // 소스 스탬프
  } catch (e) { return { sessions: [], warnings: [{ line: 0, reason: `${a.id} 수집 실패: ${e.message}` }] }; }  // 어댑터별 격리
}));
const parsed = { sessions: parsedList.flatMap((p) => p.sessions), warnings: parsedList.flatMap((p) => p.warnings) };
```
- **sourceMeta**: `new Map(adapters.map((a) => [a.id, { displayName: a.displayName, providesCost: a.providesCost }]))`. buildAnalysis가 `analyze(parsed.sessions, analyzeOpts, sourceMeta)`로 전달.
- **buildStandup**: 위 수집(기본 claude-only) 후 기존대로. cursor 안 들어옴 → 메트릭 블록 동작·숫자 불변.
- **production 멀티소스**: `cli.ts` cmdAnalyze·cmdPortrait, `mcp/server.ts` analyze 도구가 `opts.adapters = ANALYSIS_ADAPTERS` 명시 주입. (standup/hook/MCP standup은 미주입 → claude-only.)
- ⚠️ **결정성**: 코어 기본이 claude-only라, 옵션만 주는 테스트(`buildAnalysis({start})` 등)는 절대 실 Cursor DB를 안 읽는다. 멀티소스 테스트는 `adapters:[fake]` 주입. (E3 `sessionFiles` 라우팅 불필요해짐 — 기본이 이미 claude-only.)

## 8. 분석 — `src/core/analysis.ts`
`UsageAnalysis`에 **옵셔널** 추가(기존 픽스처 7개 무변경):
```typescript
export interface ToolBucket { source: string; displayName: string; sessions: number; costUsd: number; costKnown: boolean; }
  byTool?: ToolBucket[];
```
`analyze(sessions, opts = {}, sourceMeta = new Map())`:
- **costKnown 판정**: `const known = (s) => sourceMeta.get(s.source ?? "")?.providesCost ?? true;`(맵에 없으면 true — 기존 단일소스 동작 보존).
- **기존 롤업은 cost-known 세션만**: `aggregate(rangeSessions.filter(known))` → byModel·byDay·byHourKst·byProject·busiestDay·totals·hasUnknownModel 전부 cost-known 기준(Cursor의 unknown 모델·0토큰이 절대 섞이지 않음 → "단가 미상" 오염·유령 행 없음).
- **byTool은 전 소스**: rangeSessions를 `source ?? "(unknown)"`로 그룹 → 각 `{source, displayName(sourceMeta||source), sessions: count, costUsd: known?aggregate(g).totalCostUsd:0, costKnown: known}`. 세션 수 desc 정렬. (cost-known 소스가 1개뿐이면 byTool 길이 1.)
- `range`는 전 소스 기준(Cursor 활동일도 기간에 반영). `displayTokens`는 ToolBucket에서 **제외**(MVP 미렌더 — YAGNI). `analyze` 3번째 인자는 옵셔널 → `test/analysis.test.ts`의 2-인자 호출 불변.

## 9. 렌더
**portrait.ts "도구별 사용"** — 하드코딩 행 + "(E4)" 문구 제거, byTool 순회 + fallback + 캐비엇:
```
## 도구별 사용
| 도구 | 세션 | 추정 비용 |
|------|------|-----------|
| Claude Code | 23 | 약 $3,434.04 |
| Cursor | 12 | 미상 |
_세션 정의는 도구마다 다릅니다: Claude Code=세션 로그, Cursor=대화(composer)._
```
- 행: `| {displayName} | {sessions} | {costKnown ? formatMoney(costUsd) : "미상"} |`.
- **fallback(byTool 없음/빈 + sessions>0)**: 단일 `| Claude Code | totals.sessions | formatMoney(totals.costUsd) |` 행(기존 픽스처·단일소스 호환). sessions===0이면 기존 빈 상태.

**render.ts `renderAnalysis`**(= MCP analyze가 호출) — 문서에 섹션 추가(byTool fallback 동일):
```
## 도구별 사용
- Claude Code: 세션 23 · 약 $3,434.04
- Cursor: 세션 12 · 비용 미상
```

## 9b. node:sqlite 운영 세부
- **타입**: `@types/node ^22`(node:sqlite `sqlite.d.ts` 포함) 필요 — 안 올리면 `tsc` TS2307. 런타임은 `engines>=22`(검증은 Node 24). `npm install` 동반.
- **experimental 경고**: import 시 stderr only → CLI stdout·MCP stdio(JSON-RPC) 미오염. lazy import라 Cursor 실수집 때만.
- **락/부재**: 부재 파일은 `existsSync`로 조용히 skip(경고 0). 존재 DB는 `readOnly`(Cursor 실행 중 동시 읽기 OK); 열기/읽기 실패는 fail-soft warning. 핫패스 복사 안 함.

## 10. 에러 처리 / 폴백
- Cursor: 부재→조용한 skip / (열기·테이블부재·락·JSON·키·timestamp 손상)→warning + 부분 결과. **throw 절대 없음**. 오케스트레이터도 어댑터별 try/catch로 한 소스 실패가 타 소스를 안 막음.
