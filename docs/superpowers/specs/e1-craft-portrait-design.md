# E1 — AI craft 초상 (Craft Portrait) — Design

Generated by /brainstorming on 2026-06-15
Branch: main | Repo: AI_Metrics_MCP | 버전: v0.2.0 (EXPANSION E1)

> 공유 가능한 'AI craft 초상' — 분석 덤프(`analyze`)와 달리 **외부 독자용·텍스트+표만·5필드로
> 큐레이션된** 정직한 스냅샷. 기존 `UsageAnalysis` 데이터를 재사용하고 렌더·인사이트만 새로 얹는다.

## 1. 목적 & 정체성

[[aimm-primary-purpose]] 1순위(개인 상향식 AI 사용 분석/셀프 어필)의 첫 공유 산출물.
"AI 모르는 사람에게 내가 AI를 어떻게 쓰는지" 보여주는 한 장. 셀프 리뷰·승진 패킷에 그대로
붙이는, 로컬 생성·본인 소유 증거.

- **`analyze`와의 차이**: `analyze`는 분석 덤프(ASCII 막대/스파크라인, 프로젝트명 노출, 셀프
  리뷰용). 초상은 **공유용**(텍스트+표만, 프로젝트명 비노출, 외부 독자용 톤).
- 척추(상속): 서술이지 평가 아님 · 양이지 실력 아님 · 로컬·마스킹 우선 · 결정적(LLM 우회).

## 4. 인사이트 계약 (`src/core/insight.ts`)

```typescript
import type { UsageAnalysis } from "./analysis.js";

export interface Insight {
  /** 표시 문장(마크다운 인라인 허용). */
  text: string;
  /** 종류 — 디버그·E2 확장용. */
  kind: "cost-concentration" | "model-focus";
}

/**
 * UsageAnalysis만으로 계산되는 결정적 관찰. 순수 함수.
 * 세션>0이면 최소 1개(model-focus)를 보장한다(완료 기준 ③).
 * ④ 시간대는 별도 렌더 섹션이므로 여기 포함하지 않는다(중복 방지).
 */
export function deriveInsights(a: UsageAnalysis): Insight[];
```

**알고리즘(결정적):**
1. `a.totals.sessions === 0` → `[]` 반환(빈 상태는 renderPortrait가 처리).
2. **cost-concentration**(조건부): `a.busiestDay`가 있고 `a.totals.costUsd > 0`일 때
   `share = busiestDay.costUsd / totals.costUsd`. `share >= 0.2`이면
   `"비용의 약 {round(share*100)}%가 단 하루({busiestDay.date})에 집중됐습니다."` 추가.
3. **model-focus**(항상): `shortModelName`으로 패밀리 합산 후 토큰 비중 최대 패밀리 `top`.
   `"토큰의 {round(top.tokenShare*100)}%를 {top.name}로 썼습니다{단일집중?}."`
   (`top.tokenShare >= 0.8`이면 ` (단일 모델 집중)` 부기). 동률은 이름 알파벳 우선.
4. 반환 순서: `[cost-concentration?, model-focus]` (cost가 더 흥미로워 앞). 항상 ≥1.

## 5. 렌더 계약 (`src/core/portrait.ts`)

```typescript
import type { UsageAnalysis } from "./analysis.js";

export interface PortraitOptions {
  author?: string;
  /** "· 생성 <date>" 라벨. CLI가 KST 오늘 날짜 주입(순수·테스트 위해 인자로). 없으면 생략. */
  generatedDate?: string;
}

export function renderPortrait(a: UsageAnalysis, opts?: PortraitOptions): string;
```

**문서 구조(텍스트+표만, 막대/스파크라인 금지):**

```markdown
# AI Craft 초상{ — author}
기간: {range.start} ~ {range.end} (KST){ · 생성 generatedDate}

> 내가 AI를 실제로 어떻게 쓰는지의 정직한 스냅샷. 평가가 아니라 서술입니다.

## 한눈에
| 항목 | 값 |
|------|-----|
| 활동 | {기간일수}일 중 {byDay.length}일 |
| AI 세션 | {totals.sessions}건 |
| 추정 비용 | 약 {formatMoney(totals.costUsd)}{단가미상?} |
| 주력 모델 | {topModel.name} (토큰 {pct}) |
| 프로젝트 | {byProject.length}개에 걸쳐 사용 |

## 도구별 사용
| 도구 | 세션 |
|------|------|
| Claude Code | {totals.sessions} |
_다중 도구(Cursor 등)는 후속 단계(E3/E5)에서 추가됩니다._

## 비용 요약
| 항목 | 값 |
|------|-----|
| 총 추정 | 약 {formatMoney(totals.costUsd)} |
| 가장 활발한 날 | {busiestDay.date} (약 {formatMoney(busiestDay.costUsd)}) |
| 활동일 평균 | 약 {formatMoney(totals.costUsd / byDay.length)}/일 |

## 발견
{deriveInsights → 각 항목 "- {text}"}
> 결정적 관찰. 상관이지 인과·증명이 아니며, 더 깊은 인사이트는 후속(E2).

## 시간대 패턴
주로 **{peakWindow(byHourKst)}**에 사용합니다 (세션 시작 KST 최빈 구간).

## 본인 메모
_(이 줄을 지우고, 이 기간 자랑할 작업·맥락을 직접 적으세요.)_

---
⚠️ *서술*이며 *평가* 아님. 토큰·세션은 양이지 실력이 아닙니다. 비용은 추정치(단가 v{pricingVersion},
   정산액 아님). 로컬 생성·본인 소유.
```

**헬퍼·세부(결정적):**
- `formatMoney(usd)`: 천단위 콤마 + 소수 2자리. `3434.04 → "$3,434.04"`. portrait.ts 내 신규.
- `topModel`: §4.3과 동일한 패밀리 합산의 최대 토큰비중(`renderAnalysis`의 familyMap 로직 재사용 가능; 작은 헬퍼로 추출).
- `기간일수`: `range.start`~`range.end`의 일수(inclusive). 둘 다 빈 문자열이면 `byDay.length` 사용.
- `peakWindow(byHourKst)`: 최빈 시각 `m`(동률→가장 이른 시각). 인접 `m-1`,`m+1`(0~23 클립) 중 카운트가 `peak/2` 이상인 것만 포함해 `[min~max]` 범위. `dayPart(m)` 라벨과 결합 →
  예 `"밤 21~23시"`. 단일이면 `"밤 22시 전후"`. **dayPart**: 0–5 새벽 · 6–11 아침 · 12–17 오후 · 18–20 저녁 · 21–23 밤.
- `단가미상`: `a.hasUnknownModel`이면 ` (일부 모델 단가 미상)` 부기.

## 6. 마스킹 · 빈 상태 (fail-closed)

- **프로젝트명 비노출**: 초상은 `byProject`의 **개수만**(`{N}개에 걸쳐`) 쓰고 이름·슬러그는 절대
  렌더하지 않는다. 클라이언트명 누출 0. 이름별 분해는 `analyze`에만 존재.
- **스코프**: 전 프로젝트 합산(개인 전체 AI 사용 = 1순위 목적과 일치). 단 이름이 안 보이므로 공유 안전.
- **빈 상태**: `totals.sessions === 0` → 헤더 + `"이 기간에 기록된 AI 세션이 없습니다."` + 정직성
  푸터만(renderAnalysis 빈 경로와 동형). 5필드 표는 생략.

## 7. CLI (`src/cli.ts`)

- 신규 명령 `portrait`. 플래그: `--start` `--end` `--author` `--sessions`(반복). **`--repo` 없음**
  (작업 성격은 초상 5필드 아님 — YAGNI).
- 동작: `buildAnalysis({start, end, sessionFiles})` 재사용 → `renderPortrait(analysis, { author,
  generatedDate: toKstDateString(new Date()) })`를 stdout에. warnings는 stderr(analyze와 동형).
- `usage()`에 portrait 블록 추가:
  `aimm portrait [옵션]   공유용 AI craft 초상(텍스트+표) 생성` + `--start/--end/--author/--sessions` 줄.

## 8. 테스트

**`test/insight.test.ts`** (순수 함수):
- cost-concentration: busiest가 총비용의 ≥20%인 픽스처 → 해당 문장 포함; <20% → 미포함.
- model-focus: 단일 모델 100% → "토큰의 100%를 Opus로 (단일 모델 집중)"; 혼합 → 최대 패밀리 선택.
- `sessions === 0` → `[]`.

**`test/portrait.test.ts`**:
- 5필드 헤더 존재: `## 도구별 사용` · `## 비용 요약` · `## 발견` · `## 시간대 패턴` · `## 본인 메모`.
- **막대 없음**: 출력에 `█` 미포함.
- **프로젝트명 누출 없음**: `byProject`에 `"turbo-pra"` 넣은 픽스처 → 출력에 `"turbo-pra"` 미포함, 대신 `"개에 걸쳐"` 포함.
- 정직성 푸터 포함(`*서술*이며 *평가*`).
- 빈 상태: `sessions === 0` 픽스처 → "기록된 AI 세션이 없습니다", `## 도구별` 미포함.
- `peakWindow`: 손수 만든 byHourKst(22시 피크 + 21·23 인접 고빈도) → `"밤 21~23시"` 검증.

기존 패턴(render/analysis.test.ts) 따름. `npx tsc --noEmit` 클린.

## 9. 완료 기준

- `aimm portrait`가 5필드(도구별 세션·비용·발견≥1·시간대 한 줄·본인 메모 자리)를 텍스트+표로 출력.
- 막대/스파크라인 문자 없음. 프로젝트명 누출 없음.
- `--start/--end`로 기간 좁힘. 세션 0이면 graceful 빈 초상.
- 전체 테스트 그린(기존 96 + 신규) · `tsc --noEmit` 클린.
