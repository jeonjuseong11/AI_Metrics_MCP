## 1. 정체성 & CEO 플랜 재구성

원래 CEO 플랜 E2 = "git 커밋↔AI 생산성 상관"(디버깅→fix 등). 그러나 사용자 의도는
**"먼저 내가 어떻게 쓰는지(자기 인식)에 초점, 고도화하며 나중에 설득 문서까지"**.
- 따라서 E2를 **사용 패턴 발견**으로 재구성한다. git 생산성 상관(=설득/평가 영역)은
  **후속 단계**로 미룬다([[aimm-primary-purpose]]: 분석 → 셀프 어필 궤적).
- "디버깅"은 세션 메트릭으로 측정 불가(내용 라벨 없음)이라 채택 안 함.
- 척추 상속: 서술이지 평가 아님 · 양이지 실력 아님 · 결정적 · 로컬.

## 4. `day.ts` 공유 헬퍼 (DRY)

```typescript
/** 두 KST 날짜("YYYY-MM-DD") 사이 inclusive 일수. start<=end 가정. */
export function daysBetweenInclusive(start: string, end: string): number {
  const ms = Date.parse(`${end}T00:00:00Z`) - Date.parse(`${start}T00:00:00Z`);
  return Math.round(ms / 86400000) + 1;
}
```
`portrait.ts`의 기존 `periodDays`는 이 헬퍼를 쓰도록 바꾼다(빈 문자열이면 `byDay.length` 폴백 유지).

## 5. 패턴 계약 (`src/core/patterns.ts`)

`insight.ts`의 `Insight.kind`에 추가: `"session-cadence" | "weekday-rhythm" | "usage-trend" | "cost-spike"`.

```typescript
import type { UsageAnalysis } from "./analysis.js";
import type { Insight } from "./insight.js";
import { formatDuration } from "./render.js";
import { daysBetweenInclusive } from "./day.js";

export function derivePatterns(a: UsageAnalysis): Insight[];
```

**헬퍼:**
- `weekdayOf(dateStr)`: `new Date(`${dateStr}T00:00:00Z`).getUTCDay()` (0=일 … 6=토).
- `periodDays(a)`: `range` 둘 다 있으면 `daysBetweenInclusive`, 아니면 `byDay.length`.
- `isoDatePlusDays(start, n)`: `Date.parse(start+"T00:00:00Z") + n*86400000` → `YYYY-MM-DD`(UTC). (추세 중점 계산용.)
- `median(nums)`: 정렬 후 중앙값(짝수면 두 중앙의 평균). 빈 배열 0.

**축별 알고리즘(순서대로 emit; 가드 미달 시 생략):**

1. **session-cadence** (sessions>0 — 항상):
   - `avg = formatDuration(totals.durationMs / totals.sessions)`; `days = periodDays(a)`; `ratio = byDay.length/days`.
   - 한정사: `ratio < 0.3` → "며칠에 몰아서", `ratio > 0.6` → "꾸준히", else 없음.
   - text: `평균 세션 약 {avg}, 기간 {days}일 중 {byDay.length}일 활동{한정사 ? " — {한정사} 작업하는 편입니다." : "입니다."}`

2. **weekday-rhythm** (`byDay.length >= 3`):
   - `weekend = Σ sessions where weekdayOf∈{0,6}`; `share = weekend/Σsessions`.
   - `share <= 0.1` → "주로 평일에 작업합니다 (주말 비중 {pct})."
   - `share >= 0.4` → "주말에도 활발합니다 (주말 비중 {pct})."
   - else → 세션 최다 요일 `["일","월","화","수","목","금","토"][argmax]`(동률→이른 인덱스=일요일부터) → "가장 활발한 요일은 {요일}요일입니다."

3. **usage-trend** (`periodDays >= 6` AND 전·후반 둘 다 세션>0):
   - `mid = isoDatePlusDays(start, floor(days/2))`. `first = Σ sessions where date < mid`; `second = Σ where date >= mid`.
   - 둘 중 0이면 생략. `r = second/first`.
   - `r >= 1.3` → "후반부 사용이 전반부의 약 {r.toFixed(1)}배 — 최근 더 자주 씁니다."
   - `r <= 0.77` → "사용이 줄고 있습니다 (후반부가 전반부의 약 {r.toFixed(1)}배)."
   - else → "사용이 대체로 꾸준합니다."

4. **cost-spike** (`byDay.length >= 4` AND `median(costs) > 0`):
   - `med = median(byDay.costUsd)`; `spikes = byDay where costUsd > 2*med`, 비용 내림차순 상위 2개 날짜.
   - 있으면: `maxR = max(costUsd/med)` → "비용이 튄 날: {dates.join(', ')} (평소 중앙값 ${med.toFixed(2)}의 최대 약 {maxR.toFixed(1)}배)."

## 6. 합성 계약 (`src/core/findings.ts`)

```typescript
import type { UsageAnalysis } from "./analysis.js";
import { deriveInsights, type Insight } from "./insight.js";
import { derivePatterns } from "./patterns.js";

const MAX_FINDINGS = 5;

export function deriveFindings(a: UsageAnalysis): Insight[];
```
- `pattern = derivePatterns(a)`; `mini = deriveInsights(a)`.
- **중복정리**: `pattern`에 `cost-spike`가 있으면 `mini`에서 `cost-concentration` 제거(비용 이중 표기 방지).
- 순서: `[...pattern, ...miniFiltered]`. 상한 `MAX_FINDINGS`로 자른다.

## 7. `portrait.ts` 변경

- `## 발견` 루프가 `deriveFindings(a)` 호출(기존 `deriveInsights` 대신).
- 발견 노트를 갱신: 기존 `> 결정적 관찰. … 더 깊은 인사이트는 후속(E2).` →
  `> 결정적 관찰(활동일 {byDay.length}일 기준). 추세·상관 서술이지 통계적 단정·인과가 아닙니다.`
- `import { deriveFindings } from "./findings.js";` (deriveInsights import 제거).

## 8. `## 발견` mockup (E2 적용, 예시 — 생성 시 실계산)
```markdown
## 발견
- 평균 세션 약 4h, 기간 36일 중 11일 활동 — 며칠에 몰아서 작업하는 편입니다.
- 주로 평일에 작업합니다 (주말 비중 9%).
- 후반부 사용이 전반부의 약 1.4배 — 최근 더 자주 씁니다.
- 비용이 튄 날: 2026-05-17 (평소 중앙값 $X의 최대 약 2.4배).
- 토큰의 100%를 Opus로 썼습니다 (단일 모델 집중).
> 결정적 관찰(활동일 11일 기준). 추세·상관 서술이지 통계적 단정·인과가 아닙니다.
```

## 9. 정직성 (작은 n)

- 가드(활동일 ≥3/≥4, 기간 ≥6일)로 데이터 부족 시 **침묵**(거짓 패턴 방지).
- 발견 노트에 "활동일 N일 기준" 명시 + "통계적 단정·인과 아님".
- 상한 5개로 발견 과잉 방지.

## 10. 테스트

**`test/patterns.test.ts`**:
- cadence: sessions>0 픽스처 → "평균 세션"·"활동" 포함. 활동일 비율 낮은 픽스처 → "몰아서".
- weekday: 주말 비중 낮은 byDay(평일 날짜들) → "주로 평일"; 주말 높은 → "주말에도"; 중간 → 최다 요일.
- trend: 후반부 세션 많은 7일+ 픽스처 → "더 자주"; 적은 → "줄고"; 비슷 → "꾸준". `byDay.length<` 가드 미달 → 해당 축 없음.
- cost-spike: 한 날만 큰 4일+ 픽스처 → "튄 날" + 배수. 활동일 3 이하 → 없음.

**`test/findings.test.ts`**:
- cost-spike 있으면 cost-concentration 빠짐. 상한 5. 순서(패턴 먼저).

**`test/portrait.test.ts`** (수정):
- 다일(多日) 픽스처 → 발견에 "평균 세션"(cadence) 등장. 기존 5필드·footer 단언 유지.
